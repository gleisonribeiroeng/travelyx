Create `transport.mapper.ts` following the car.mapper.ts pattern exactly:

1. Define `ExternalRoute` interface with ALL fields optional (API shape is hypothetical):
   - `id?: number | string`
   - `type?: string` (or `mode?: string`) ã maps to Transport.mode union
   - `origin?: string` (or `from?: string`) ã origin city
   - `destination?: string` (or `to?: string`) ã destination city
   - `departure?: string` (or `departureTime?: string`) ã ISO 8601 datetime
   - `arrival?: string` (or `arrivalTime?: string`) ã ISO 8601 datetime
   - `duration?: string | number` ã ISO 8601 "PT2H30M" or human-readable "2h 30m" or raw minutes
   - `price?: number` (or `total_price?: number`)
   - `currency?: string`
   - `url?: string` (or `booking_url?: string`) ã external provider link
   - `operator?: string` (or `carrier?: string`) ã provider name for link.provider

2. Define `TransportSearchParams` interface:
   - `origin: string` ã plain text city name
   - `destination: string` ã plain text city name
   - `departureDate: string` ã YYYY-MM-DD format
   - `currency?: string` ã default 'USD'

3. Create `TransportMapper` class (`@Injectable({ providedIn: 'root' })`):
   - Does NOT implement Mapper interface (two-param mapResponse like CarMapper)
   - `mapResponse(raw: ExternalRoute, params: TransportSearchParams): Transport`
     - `id`: `String(raw.id || crypto.randomUUID())`
     - `source`: `'transport'`
     - `addedToItinerary`: `false`
     - `mode`: `this.mapMode(raw.type || raw.mode || '')`
     - `origin`: `raw.origin || raw.from || params.origin`
     - `destination`: `raw.destination || raw.to || params.destination`
     - `departureAt`: `raw.departure || raw.departureTime || ''`
     - `arrivalAt`: `raw.arrival || raw.arrivalTime || ''`
     - `durationMinutes`: `this.parseDuration(raw.duration || '0')`
     - `price`: `{ total: raw.price || raw.total_price || 0, currency: raw.currency || 'USD' }`
     - `link`: `{ url: raw.url || raw.booking_url || '#', provider: raw.operator || raw.carrier || 'Transport' }`
   - Private `mapMode(type: string): 'bus' | 'train' | 'ferry' | 'other'` ã case-insensitive substring matching:
     - includes('bus') or includes('coach') -> 'bus'
     - includes('train') or includes('rail') -> 'train'
     - includes('ferry') or includes('boat') -> 'ferry'
     - default -> 'other'
   - Private `parseDuration(duration: string | number): number`:
     - If typeof number, return as-is
     - If starts with 'PT': parse ISO 8601 (match /(\d+)H/ and /(\d+)M/)
     - Else: parse human-readable (match /(\d+)\s*h/i and /(\d+)\s*m/i)
     - Return total minutes

Then create `transport-api.service.ts` following car-api.service.ts exactly:

1. `@Injectable({ providedIn: 'root' })` class extending `BaseApiService`
2. `private readonly mapper = inject(TransportMapper)`
3. `constructor() { super('transport'); }` ã maps to `/api/transport` proxy endpoint
4. `searchTransport(params: TransportSearchParams): Observable<ApiResult<Transport[]>>`:
   - Call `this.get<any>('/api/v1/transport/search', { origin: params.origin, destination: params.destination, date: params.departureDate, currency: params.currency || 'USD' })`
   - Pipe through `withBackoff()`
   - `map()` response: extract results array with fallback chain `response.data?.routes || response.data?.results || response.data || []`, ensure Array.isArray, map each through `this.mapper.mapResponse(r, params)`
   - Return `{ data: mapped, error: null }` on success
   - `catchError((error: AppError) => of({ data: [], error }))` on failure
5. Re-export TransportSearchParams: `export type { TransportSearchParams } from './transport.mapper';`
6. Add JSDoc noting endpoint and params are HYPOTHETICAL ã must be updated when provider is selected

Import from existing modules:
- `BaseApiService` from `./base-api.service`
- `Transport` from `../models/trip.models`
- `ApiResult` from `./api-error.utils`
- `AppError` from `./models/app-error.model`
- `withBackoff` from `./retry.utils`

NOTE: Transport is NOT a RapidAPI source. The interceptor handles it with standard X-API-Key header (not in RAPID_API_SOURCES array). This is correct ã do not modify the interceptor.
