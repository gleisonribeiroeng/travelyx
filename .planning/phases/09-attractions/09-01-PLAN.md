**attraction.mapper.ts** — Create AttractionMapper following TourMapper pattern:

1. Define external response interfaces (all fields optional for safety):
   - `GeonameResponse`: { name?, country?, lat?, lon?, population?, timezone? }
   - `RadiusFeature`: { name?, xid?, osm?, wikidata?, kind?, point?: { lon?, lat? } }
   - `PlaceDetails`: { xid?, name?, address?: { city?, road?, house_number? }, kinds?, wikipedia?, url?, image?, preview?: { source?, width?, height? }, point?: { lon?, lat? } }
   - `AttractionSearchParams`: { city: string }

2. Create `@Injectable({ providedIn: 'root' })` class AttractionMapper with:
   - `mapResponse(raw: RadiusFeature, params: AttractionSearchParams): Attraction` — maps radius feature to Attraction:
     - `id`: raw.xid || crypto.randomUUID()
     - `source`: 'attractions'
     - `addedToItinerary`: false
     - `name`: raw.name || 'Unnamed Attraction'
     - `description`: '' (not available in radius response)
     - `location`: { latitude: raw.point?.lat || 0, longitude: raw.point?.lon || 0 }
     - `city`: params.city
     - `category`: extractPrimaryCategory(raw.kind) — private method
     - `link`: null (not available in radius; enriched later)
   - `enrichWithDetails(attraction: Attraction, details: PlaceDetails): Attraction` — merges detail data:
     - Spread attraction, override:
     - `description`: details.wikipedia ? `Learn more: ${details.wikipedia}` : (details.url ? 'Official site available.' : '')
     - `link`: details.url ? { url: details.url, provider: 'Official' } : null — THIS IS THE NULLABLE LINK PATTERN
   - Private `extractPrimaryCategory(kindString: string | undefined): string`:
     - Split by comma, take first, replace underscores with spaces, capitalize first letter
     - Default to 'Attraction' if empty/undefined

3. Does NOT implement Mapper interface (two-param mapResponse, same pattern as TourMapper, TransportMapper, CarMapper).

4. Import Attraction from '../models/trip.models'.

**attraction-api.service.ts** — Create AttractionApiService following TourApiService pattern:

1. `@Injectable({ providedIn: 'root' })` class extending BaseApiService.
2. Constructor: `super('attractions')` — matches ApiConfigService key 'attractions'.
3. `private readonly mapper = inject(AttractionMapper)`.
4. Re-export: `export type { AttractionSearchParams } from './attraction.mapper';`

5. Three private helper methods:
   - `getGeoname(cityName: string): Observable<ApiResult<{ lat: number; lon: number } | null>>`:
     - `this.get<GeonameResponse>('/0.1/en/places/geoname', { name: cityName })` — BaseApiService.get() auto-adds query params
     - Pipe: withBackoff(), map to { data: { lat: response.lat, lon: response.lon }, error: null }
     - catchError returns { data: null, error }
     - NOTE: OpenTripMap uses `apikey` query param. The interceptor sends X-API-Key header. If this doesn't work at runtime, the `apikey` param can be manually added here. For now, rely on interceptor + proxy forwarding.

   - `getRadius(lat: number, lon: number, radius: number = 5000): Observable<ApiResult<RadiusFeature[]>>`:
     - `this.get<any>('/0.1/en/places/radius', { lat: lat.toString(), lon: lon.toString(), radius: radius.toString(), limit: '20', rate: '2' })`
     - rate: '2' filters for named attractions only (OpenTripMap rate filter: 1=unnamed, 2=named, 3=famous)
     - Pipe: withBackoff(), map with defensive extraction: `response.features || (Array.isArray(response) ? response : [])`, then filter out empty names (`.filter(f => f.name && f.name.trim() !== '')`), return { data: features, error: null }
     - catchError returns { data: [], error }

   - `enrichWithDetails(features: RadiusFeature[], params: AttractionSearchParams): Observable<ApiResult<Attraction[]>>`:
     - Map each feature to basic Attraction via `this.mapper.mapResponse(feature, params)`
     - For each attraction's xid, call `this.get<PlaceDetails>('/0.1/en/places/xid/' + feature.xid).pipe(withBackoff(), catchError(() => of(null)))`
     - Use `forkJoin()` to fetch all details in parallel
     - Merge each detail into its corresponding attraction via `this.mapper.enrichWithDetails()` (skip if detail is null)
     - Return `{ data: enrichedAttractions, error: null }`
     - Wrap entire forkJoin in catchError returning `{ data: basicAttractions, error }` as graceful fallback (show basic results if enrichment fails)
     - If features array is empty, return `of({ data: [], error: null })` immediately

6. Public method `searchAttractions(params: AttractionSearchParams): Observable<ApiResult<Attraction[]>>`:
   - Chain: getGeoname(params.city) -> switchMap to getRadius(lat, lon) -> switchMap to enrichWithDetails(radiusFeatures, params)
   - If geoname returns null data (city not found), return `of({ data: [], error: geonameResult.error })` immediately
   - If radius returns error, return `of({ data: [], error: radiusResult.error })` immediately
   - This three-step chained observable pattern is unique to attractions (other services use single endpoint)

Import from existing modules: BaseApiService, ApiResult, AppError, withBackoff, Attraction, forkJoin, of, Observable, switchMap, map, catchError.
