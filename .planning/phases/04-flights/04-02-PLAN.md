**Step 2: Replace SearchComponent TypeScript**

Replace `triply/src/app/features/search/search.component.ts` entirely. The new component must:

1. **Imports array:** `[MATERIAL_IMPORTS, ReactiveFormsModule, CommonModule, MatAutocompleteModule]`
   - NOTE: MatAutocompleteModule is also in MATERIAL_IMPORTS after Step 1, but explicit import ensures it's available even before rebuild. Actually since Step 1 adds it, just `[MATERIAL_IMPORTS, ReactiveFormsModule, CommonModule]` is sufficient.

2. **Injected services:**
   - `private readonly flightApi = inject(FlightApiService)`
   - `private readonly tripState = inject(TripStateService)`
   - `private readonly snackBar = inject(MatSnackBar)`

3. **Form controls (reactive form):**
   - `originControl = new FormControl<AirportOption | null>(null, [Validators.required, this.airportValidator()])`
   - `destinationControl = new FormControl<AirportOption | null>(null, [Validators.required, this.airportValidator()])`
   - `flightSearchForm = new FormGroup({ origin: this.originControl, destination: this.destinationControl, dateRange: new FormGroup({ departure: new FormControl<Date | null>(null, Validators.required), return: new FormControl<Date | null>(null) }), passengers: new FormControl(1, [Validators.required, Validators.min(1), Validators.max(9)]) })`

4. **Autocomplete observables:**
   - `filteredOrigins$` = `this.originControl.valueChanges.pipe(debounceTime(300), distinctUntilChanged(), filter(v => typeof v === 'string' && v.length >= 2), switchMap(keyword => this.flightApi.searchAirports(keyword as string)))`
   - `filteredDestinations$` = same pattern for `destinationControl`

5. **Search state signals:**
   - `searchResults = signal<Flight[]>([])`
   - `isSearching = signal(false)`
   - `filterType = signal<'all' | 'direct' | 'stopovers'>('all')`
   - `sortBy = signal<'price' | 'duration' | 'stops'>('price')`

6. **Computed signals:**
   - `filteredFlights = computed(() => { ... })` — filter by filterType, then sort by sortBy:
     - Filter: 'all' returns all, 'direct' returns stops === 0, 'stopovers' returns stops > 0
     - Sort: 'price' by price.total ascending, 'duration' by durationMinutes ascending, 'stops' by stops ascending

7. **Date getters:**
   - `get minDepartureDate(): Date { return new Date(); }`
   - `get minReturnDate(): Date { return this.flightSearchForm.value.dateRange?.departure || new Date(); }`

8. **Methods:**
   - `searchFlights(): void` — validates form, extracts values, converts departure Date to YYYY-MM-DD string via `date.toISOString().split('T')[0]`, calls `flightApi.searchFlights()`, sets isSearching, handles result/error via subscribe with finalize
   - `addToItinerary(flight: Flight): void` — calls `tripState.addFlight({...flight, addedToItinerary: true})`, shows snackbar "Flight added to itinerary"
   - `displayAirport(airport: AirportOption | null): string` — returns `airport ? \`${airport.iataCode} - ${airport.cityName}\` : ''`
   - `formatDuration(minutes: number): string` — returns `\`${Math.floor(minutes / 60)}h ${minutes % 60}m\``
   - `setFilter(value: string): void` — sets filterType signal (cast to union type)
   - `setSortBy(value: string): void` — sets sortBy signal (cast to union type)
   - `private airportValidator(): ValidatorFn` — returns null if value is null (let required handle), returns `{ invalidAirport: true }` if typeof value === 'string' or value has no iataCode property

Import from:
- `@angular/core`: Component, signal, computed, inject
- `@angular/forms`: FormControl, FormGroup, Validators, ValidatorFn, AbstractControl, ValidationErrors, ReactiveFormsModule
- `@angular/common`: CommonModule
- `@angular/material/snack-bar`: MatSnackBar

- `../core/services/trip-state.service`: TripStateService

- RxJS: Observable, debounceTime, distinctUntilChanged, filter, switchMap, finalize

**Step 3: Replace SearchComponent HTML template**

Replace `triply/src/app/features/search/search.component.html` with:

1. **Header section:**
   - `<div class="view-header">` with `<h1>Search Flights</h1>` and `<p>Find the best flights for your trip</p>`

2. **Search form card:**
   - Wrap in `<mat-card class="search-form-card">` with `<mat-card-content>`
   - `<form [formGroup]="flightSearchForm" (ngSubmit)="searchFlights()">`
   - **Row 1: Origin + Destination** (two `<mat-form-field>` side by side in a `.form-row`)
     - Origin: `<input matInput [formControl]="originControl" [matAutocomplete]="autoOrigin" placeholder="City or airport">`, with `<mat-autocomplete #autoOrigin [displayWith]="displayAirport">` showing `@for (airport of filteredOrigins$ | async; track airport.iataCode)` with `<mat-option [value]="airport">{{airport.iataCode}} - {{airport.name}}, {{airport.cityName}}</mat-option>`
     - Same pattern for destination with `autoDestination` and `filteredDestinations$`
     - Validation errors: `<mat-error>` for required and invalidAirport
   - **Row 2: Date range + Passengers** (in `.form-row`)
     - Date range: `<mat-form-field>` with `<mat-date-range-input [rangePicker]="picker" [min]="minDepartureDate">` containing `<input matStartDate formControlName="departure">` and `<input matEndDate formControlName="return">`. Add `<mat-datepicker-toggle matIconSuffix [for]="picker">` and `<mat-date-range-picker #picker>`. Wrap the date range inputs in a `formGroupName="dateRange"` div.
     - Passengers: `<mat-form-field>` with `<mat-select formControlName="passengers">` and `<mat-option>` for 1-9 adults
   - **Submit button:** `<button mat-raised-button color="primary" type="submit" [disabled]="flightSearchForm.invalid || isSearching()">` with search icon and "Search Flights" / "Searching..." text based on `isSearching()` signal. Show `<mat-spinner diameter="20">` when searching.

3. **Results section** — only show when `searchResults().length > 0`:
   - **Results header:** `<h2>{{filteredFlights().length}} of {{searchResults().length}} flights</h2>`
   - **Controls row** (`.results-controls`):
     - Filter chips: `<mat-chip-listbox (change)="setFilter($event.value)" [value]="filterType()">` with three `<mat-chip-option>` for 'all', 'direct', 'stopovers' — show count for each
     - Sort dropdown: `<mat-form-field>` with `<mat-select (selectionChange)="setSortBy($event.value)" [value]="sortBy()">` — options: "Price (lowest)", "Duration (shortest)", "Stops (fewest)"
   - **Flight result cards** — `@for (flight of filteredFlights(); track flight.id)`:
     - `<mat-card class="flight-card">`
     - `<mat-card-content>`:
       - `.flight-header` row: route (`flight.origin` arrow `flight.destination`) and price (`flight.price.total | currency:flight.price.currency`)
       - `.flight-details` row: duration (use `formatDuration(flight.durationMinutes)`), airline + flight number, stops (Direct or N stop(s)) — each with a `<mat-icon>`
       - `.flight-times` row: departure datetime and arrival datetime using `date:'short'` pipe
     - `<mat-card-actions>`:
       - "Add to Itinerary" button: `<button mat-button color="primary" (click)="addToItinerary(flight)">` with add icon
       - Provider link: `<a mat-button [href]="flight.link.url" target="_blank" rel="noopener noreferrer">` with open_in_new icon and "View on {{flight.link.provider}}"

4. **Empty state** — show when `searchResults().length === 0 && !isSearching()` and form has been submitted (track with a `hasSearched` signal set to true after first search):
   - `<mat-card class="empty-state">` with flight_takeoff icon and "No flights found. Try different dates or airports."

Use Angular's `@if`, `@for` control flow (NOT `*ngIf`, `*ngFor`) — Angular 21 uses the new template syntax. Use `@for` with `track` clause.
