Create `triply/src/app/core/api/car.mapper.ts` with:

1. **BookingComCar interface** (the external API shape from booking-com15.p.rapidapi.com):
   ```typescript
   export interface BookingComCar {
     vehicle_id?: number | string;
     vehicle_name?: string;
     vehicle_category?: string;    // e.g. "Economy", "Compact", "SUV"
     group?: string;               // Alternative field for vehicle category
     price?: number;
     total_price?: number;
     currency?: string;
     url?: string;
     deep_link?: string;
     supplier?: string;            // e.g. "Hertz", "Avis"
     supplier_name?: string;
   }
   ```
   Mark ALL fields as optional (`?`) because the API response format is hypothetical and may vary. The mapper uses fallback chains for every field.

2. **CarSearchParams interface** (exported, used by CarApiService and CarMapper):
   ```typescript
   export interface CarSearchParams {
     pickupLocation: string;     // City name or airport code (plain text)
     dropoffLocation: string;    // City name or airport code (plain text)
     pickupAt: string;           // ISO 8601 datetime: 2026-03-15T10:00:00
     dropoffAt: string;          // ISO 8601 datetime: 2026-03-20T10:00:00
     driverAge: number;          // Default 30, range 18-99
     currency?: string;          // Default 'USD'
   }
   ```

3. **CarMapper class** -- `@Injectable({ providedIn: 'root' })`:
   - `mapResponse(raw: BookingComCar, params: CarSearchParams): CarRental` -- transforms one BookingComCar to the canonical CarRental model:
     - `id` = `String(raw.vehicle_id || crypto.randomUUID())` -- use vehicle_id if available, fallback to UUID
     - `source` = `'carRental'`
     - `addedToItinerary` = `false`
     - `vehicleType` = `raw.vehicle_category || raw.group || 'Unknown'`
     - `pickUpLocation` = `params.pickupLocation` -- echo from search params
     - `dropOffLocation` = `params.dropoffLocation` -- echo from search params
     - `pickUpAt` = `params.pickupAt` -- echo ISO 8601 datetime
     - `dropOffAt` = `params.dropoffAt` -- echo ISO 8601 datetime
     - `price` = `{ total: raw.price || raw.total_price || 0, currency: raw.currency || 'USD' }`
     - `link` = `{ url: raw.url || raw.deep_link || 'https://www.booking.com', provider: raw.supplier || raw.supplier_name || 'Booking.com' }`

   NOTE: Unlike HotelMapper which implements `Mapper<BookingComHotel, Stay>` formally, CarMapper does NOT need to implement the Mapper interface because its mapResponse takes two parameters (raw + params) which deviates from the single-parameter Mapper interface. This is the same approach HotelMapper uses (it adds optional extra params). However, for CarMapper the params are NOT optional (pickup/dropoff locations come from search params, not from the API response). So declare CarMapper as a plain injectable class without `implements Mapper<...>`. This keeps it honest about its signature.

Import from:
- `@angular/core`: Injectable
- `../models/trip.models`: CarRental

**Critical: Do NOT convert datetime strings to Date objects in the final CarRental model.** pickUpAt/dropOffAt stay as ISO 8601 strings per project decision (03-01).

**Step 1: Create CarApiService**

Create `triply/src/app/core/api/car-api.service.ts`:

```typescript
@Injectable({ providedIn: 'root' })
export class CarApiService extends BaseApiService {
  private readonly mapper = inject(CarMapper);

  constructor() {
    super('carRental');
  }

  searchCars(params: CarSearchParams): Observable<ApiResult<CarRental[]>> { ... }
}
```

The `searchCars` method:
- Call `this.get<any>('/api/v1/cars/searchCarRentals', { ... })` with these query parameters:
  ```typescript
  {
    pick_up_location: params.pickupLocation,
    drop_off_location: params.dropoffLocation,
    pick_up_datetime: params.pickupAt,
    drop_off_datetime: params.dropoffAt,
    driver_age: params.driverAge,
    currency: params.currency || 'USD',
  }
  ```
- NOTE: The endpoint path `/api/v1/cars/searchCarRentals` and parameter names are HYPOTHETICAL (based on the booking-com15 hotel pattern `/api/v1/hotels/searchHotels`). They MUST be verified at implementation time via the RapidAPI dashboard. The mapper is designed to handle response format uncertainty with safe fallback chains.
- BaseApiService.get() prepends the endpoint URL (`/api/cars` from getEndpoint('carRental')), so calling `this.get('/api/v1/cars/searchCarRentals')` produces URL `http://localhost:4200/api/cars/api/v1/cars/searchCarRentals`. The dev proxy matches the `/api/cars` prefix and rewrites it to `https://booking-com15.p.rapidapi.com/api/v1/cars/searchCarRentals`.
- Pipe through:
  - `withBackoff()` for rate-limit retry
  - `map()` to extract results defensively:
    ```typescript
    map((response): ApiResult<CarRental[]> => {
      const results = response.data?.result || response.data?.cars || response.data || [];
      const cars = Array.isArray(results) ? results : [];
      return {
        data: cars.map((car: BookingComCar) => this.mapper.mapResponse(car, params)),
        error: null,
      };
    })
    ```
  - `catchError((error: AppError): Observable<ApiResult<CarRental[]>> => of({ data: [], error }))` -- per-source error isolation
