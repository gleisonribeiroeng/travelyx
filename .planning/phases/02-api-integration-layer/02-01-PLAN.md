  <action>
Create three functional interceptors following the Angular 21 `HttpInterceptorFn` pattern:

1. **API Key Interceptor** (`interceptors/api-key.interceptor.ts`):
   - Export `API_SOURCE = new HttpContextToken<string | null>(() => null)` — callers set this to identify which API key to inject.
   - Export `const apiKeyInterceptor: HttpInterceptorFn = (req, next) => { ... }`.
   - Use `inject(ApiConfigService)` to get the config service.
   - Read `req.context.get(API_SOURCE)` — if null, pass through with `next(req)`.
   - If source exists, call `apiConfig.getKey(source)` — if key is null or empty string, pass through.
   - Otherwise clone request with `req.clone({ headers: req.headers.set('X-API-Key', key) })` and forward.
   - NOTE: Amadeus uses OAuth2 Bearer tokens (handled by proxy), so the X-API-Key header is for simpler APIs. The interceptor just adds whatever key is configured — the proxy handles Amadeus-specific auth.

2. **Error Interceptor** (`interceptors/error.interceptor.ts`):
   - Import `API_SOURCE` from `./api-key.interceptor`.
   - Export `const errorInterceptor: HttpInterceptorFn = (req, next) => { ... }`.
   - Pipe `next(req)` through `catchError((error: HttpErrorResponse) => { ... })`.
   - Inside catchError, construct an `AppError` object: `status` from `error.status`, `code` from `error.error?.code ?? 'UNKNOWN'`, `message` from `error.error?.detail ?? error.error?.message ?? error.message ?? 'An unexpected error occurred'`, `source` from `req.context.get(API_SOURCE) ?? 'unknown'`, `timestamp` as `new Date().toISOString()`.
   - Return `throwError(() => normalized)`.

3. **Loading Interceptor** (`interceptors/loading.interceptor.ts`):
   - Export `const loadingInterceptor: HttpInterceptorFn = (req, next) => { ... }`.
   - Use `inject(LoadingService)` to get loading service.
   - Call `loadingService.increment()` immediately.
   - Pipe `next(req)` through `finalize(() => loadingService.decrement())`.
   - CRITICAL: `finalize()` must be the ONLY operator in this interceptor's pipe — it fires on both complete and error, guaranteeing the counter decrements. Do NOT add catchError before finalize here.
  </action>
  <verify>Run `npx ng build` from triply/ — must compile with zero errors. Verify all three interceptor files exist and export the correct function names.</verify>
  <done>Three functional interceptors compile cleanly: apiKeyInterceptor injects keys via HttpContextToken, errorInterceptor normalizes all HTTP errors to AppError, loadingInterceptor tracks in-flight count via finalize().</done>
</task>
