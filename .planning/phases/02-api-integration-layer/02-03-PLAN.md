Create two files that feature API services will depend on:

1. **BaseApiService** (`core/api/base-api.service.ts`):
   An abstract injectable class that feature API services extend.

   - Protected fields: `http = inject(HttpClient)`, `config = inject(ApiConfigService)`.
   - Constructor accepts `protected readonly apiSource: string` (e.g., 'amadeus', 'hotel') â€” the source identifier used for HttpContextToken.
   - Protected method `get<T>(path: string, params?: Record<string, string | number | boolean>): Observable<T>`:
     - Constructs full URL from `this.config.getEndpoint(this.apiSource) + path`.
     - Creates `HttpContext` with `API_SOURCE` set to `this.apiSource`.
     - Calls `this.http.get<T>(url, { params: httpParams, context })`.
   - Protected method `post<T>(path: string, body: unknown): Observable<T>`:
     - Same URL and context construction.
     - Calls `this.http.post<T>(url, body, { context })`.

   This ensures every request from any feature service automatically gets the correct API_SOURCE for the interceptor to inject the right key, and uses the correct proxy endpoint.

   Do NOT decorate with `@Injectable()` â€” it is abstract. Concrete subclasses will be `@Injectable({ providedIn: 'root' })`.

2. **Per-source error handling utility** (`core/api/api-error.utils.ts`):
   Export a generic `ApiResult<T>` type: `{ data: T; error: null } | { data: T; error: AppError }` where the first form represents success and the second represents partial failure with fallback data.

   Export a function `withFallback<T>(fallback: T)` that returns an RxJS operator function. Usage:
   ```
   flightsService.search(params).pipe(withFallback([]))
   ```
   Implementation: returns `(source$: Observable<T>) => source$.pipe(map(data => ({ data, error: null })), catchError((err: AppError) => of({ data: fallback, error: err })))`.

   This is the per-source catchError pattern that makes observables safe for forkJoin. Each source gets its own fallback so one failure does not kill the entire page.
  </action>
