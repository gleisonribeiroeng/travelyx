Create `core/api/retry.utils.ts` with:

1. Export `RETRIABLE_STATUS_CODES = [429, 502, 503, 504]` — the HTTP status codes that warrant retry.

2. Export `DEFAULT_RETRY_CONFIG = { maxRetries: 3, initialDelay: 1000, maxDelay: 8000 }`.

3. Export function `withBackoff<T>(config?: Partial<typeof DEFAULT_RETRY_CONFIG>)` that returns a `MonoTypeOperatorFunction<T>` (an RxJS pipeable operator).

Implementation:
- Merge provided config with defaults.
- Return the RxJS `retry()` operator with this configuration:
  ```
  retry({
    count: config.maxRetries,
    delay: (error, retryIndex) => {
      // Only retry on retriable status codes
      if (RETRIABLE_STATUS_CODES.includes(error.status)) {
        const delay = Math.min(
          config.initialDelay * Math.pow(2, retryIndex),
          config.maxDelay
        );
        return timer(delay);
      }
      // Non-retriable errors (400, 401, 403, 404, etc.) fail immediately
      return throwError(() => error);
    },
  })
  ```

- Import `retry` from `rxjs/operators`, `timer` and `throwError` from `rxjs`.
- The `error` parameter will be an `AppError` (from the error interceptor) — so check `error.status`.
- Do NOT use deprecated `retryWhen` — this is the modern `retry({ count, delay })` API from RxJS 7.3+.

Usage example (documented in a JSDoc comment on withBackoff):
```typescript
this.http.get('/api/amadeus/flights').pipe(
  withBackoff(),           // default: 3 retries, 1s/2s/4s delays
  withBackoff({ maxRetries: 5 }),  // custom config
)
```

Create `core/api/search.utils.ts` with:

1. Export `DEFAULT_DEBOUNCE_MS = 400` — the default debounce interval for search forms.

2. Export function `debouncedSearch<TInput, TOutput>(searchFn: (input: TInput) => Observable<TOutput>, debounceMs?: number)` that returns an `OperatorFunction<TInput, TOutput>` (a pipeable operator for form value streams).

Implementation:
```typescript
export function debouncedSearch<TInput, TOutput>(
  searchFn: (input: TInput) => Observable<TOutput>,
  debounceMs: number = DEFAULT_DEBOUNCE_MS,
): OperatorFunction<TInput, TOutput> {
  return (source$: Observable<TInput>) =>
    source$.pipe(
      debounceTime(debounceMs),
      distinctUntilChanged((a, b) => JSON.stringify(a) === JSON.stringify(b)),
      filter((value): value is TInput => value != null),
      switchMap(value => searchFn(value)),
    );
}
```

Key design decisions:
- `debounceTime` sits BEFORE `switchMap` — this prevents request creation on every keystroke (research pitfall #5)
- `switchMap` cancels in-flight requests when a new value arrives — only the latest search matters
- `distinctUntilChanged` with JSON comparison prevents duplicate searches for the same input
- `filter` removes null/undefined values (e.g., when form is cleared)

Import `debounceTime`, `distinctUntilChanged`, `filter`, `switchMap` from `rxjs/operators`, `Observable`, `OperatorFunction` from `rxjs`.

Usage example (documented in JSDoc):
```typescript
// In search component:
this.searchForm.valueChanges.pipe(
  debouncedSearch(params => this.flightsApi.search(params)),
).subscribe(results => this.results.set(results));
```
