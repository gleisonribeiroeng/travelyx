**Step 1: Create HotelSearchComponent TypeScript**

Create `triply/src/app/features/hotel-search/hotel-search.component.ts`:

```typescript
@Component({
  selector: 'app-hotel-search',
  standalone: true,
  imports: [MATERIAL_IMPORTS, ReactiveFormsModule, CommonModule],
  templateUrl: './hotel-search.component.html',
  styleUrl: './hotel-search.component.scss',
})
export class HotelSearchComponent { ... }
```

1. **Injected services:**
   - `private readonly hotelApi = inject(HotelApiService)`
   - `private readonly tripState = inject(TripStateService)`
   - `private readonly snackBar = inject(MatSnackBar)`

2. **Form controls (reactive form):**
   - `destinationControl = new FormControl<DestinationOption | null>(null, [Validators.required, this.destinationValidator()])`
   - `hotelSearchForm = new FormGroup({ destination: this.destinationControl, checkIn: new FormControl<Date | null>(null, Validators.required), checkOut: new FormControl<Date | null>(null, Validators.required), guests: new FormControl(2, [Validators.required, Validators.min(1), Validators.max(30)]), rooms: new FormControl(1, [Validators.required, Validators.min(1), Validators.max(10)]) })`
   - NOTE: Unlike flights which use a date RANGE picker (single mat-date-range-input with start/end), hotels use TWO SEPARATE mat-datepicker fields for check-in and check-out. This is because check-in/check-out are top-level form controls (not nested in a dateRange FormGroup). Using separate pickers avoids the awkward range-picker UX for hotel dates and makes validation simpler.

3. **Autocomplete observable:**
   - `filteredDestinations$` = `this.destinationControl.valueChanges.pipe(debounceTime(300), distinctUntilChanged(), filter(v => typeof v === 'string' && (v as string).length >= 2), switchMap(keyword => this.hotelApi.searchDestinations(keyword as string)))`

4. **Search state signals:**
   - `searchResults = signal<Stay[]>([])`
   - `isSearching = signal(false)`
   - `hasSearched = signal(false)`
   - `sortBy = signal<'price' | 'rating'>('price')`

5. **Computed signals:**
   - `sortedHotels = computed(() => { ... })` — sort by sortBy:
     - 'price': sort by `pricePerNight.total` ascending
     - 'rating': sort by `rating` descending (highest first), nulls last

6. **Date getters:**
   - `get minCheckInDate(): Date { return new Date(); }`
   - `get minCheckOutDate(): Date { return this.hotelSearchForm.value.checkIn || new Date(); }`

7. **Methods:**
   - `searchHotels(): void` — validates form, extracts values, converts check-in/check-out Date objects to YYYY-MM-DD strings via `date.toISOString().split('T')[0]`, calls `hotelApi.searchHotels({ destId: destination.destId, searchType: destination.searchType, checkIn, checkOut, adults: guests, rooms })`, sets isSearching, handles result/error via subscribe with finalize. On error, show snackbar "Failed to search hotels. Please try again."
   - `addToItinerary(hotel: Stay): void` — calls `tripState.addStay({...hotel, addedToItinerary: true})`, shows snackbar "Hotel added to itinerary"
   - `displayDestination(dest: DestinationOption | null): string` — returns `dest ? dest.label || dest.name : ''`
   - `formatRating(rating: number | null): string` — returns `rating != null ? rating.toFixed(1) + ' / 5' : 'No rating'`
   - `renderStars(rating: number | null): string[]` — returns an array of 5 strings ('star', 'star_half', 'star_border') for template to render star icons based on rating. For example, rating 3.7 produces ['star', 'star', 'star', 'star_half', 'star_border'].
   - `setSortBy(value: string): void` — sets sortBy signal (cast to union type)
   - `private destinationValidator(): ValidatorFn` — returns null if value is null (let required handle), returns `{ invalidDestination: true }` if typeof value === 'string' or value has no destId property

Import from:
- `@angular/core`: Component, signal, computed, inject
- `@angular/forms`: FormControl, FormGroup, Validators, ValidatorFn, AbstractControl, ValidationErrors, ReactiveFormsModule
- `@angular/common`: CommonModule
- `@angular/material/snack-bar`: MatSnackBar
- `../core/material.exports`: MATERIAL_IMPORTS
- `../core/api/hotel-api.service`: HotelApiService, DestinationOption
- `../core/services/trip-state.service`: TripStateService
- `../core/models/trip.models`: Stay
- RxJS: Observable, debounceTime, distinctUntilChanged, filter, switchMap, finalize

**Step 2: Create HotelSearchComponent HTML template**

Create `triply/src/app/features/hotel-search/hotel-search.component.html`:

1. **Header section:**
   - `<div class="view-header">` with `<h1>Search Hotels</h1>` and `<p>Find the perfect accommodation for your trip</p>`

2. **Search form card:**
   - Wrap in `<mat-card class="search-form-card">` with `<mat-card-content>`
   - `<form [formGroup]="hotelSearchForm" (ngSubmit)="searchHotels()">`
   - **Row 1: Destination** (full-width `<mat-form-field>`)
     - `<input matInput [formControl]="destinationControl" [matAutocomplete]="autoDestination" placeholder="City or destination">`
     - `<mat-icon matPrefix>location_on</mat-icon>`
     - `<mat-autocomplete #autoDestination [displayWith]="displayDestination">`
       - `@for (dest of filteredDestinations$ | async; track dest.destId) { <mat-option [value]="dest">{{ dest.label }}</mat-option> }`
     - Validation errors: `<mat-error>` for required and invalidDestination ("Please select a destination from the list")
   - **Row 2: Check-in + Check-out** (two separate `<mat-form-field>` in `.form-row`)
     - Check-in: `<mat-form-field>` with `<mat-label>Check-in</mat-label>`, `<input matInput [matDatepicker]="checkInPicker" formControlName="checkIn" [min]="minCheckInDate">`, `<mat-datepicker-toggle matIconSuffix [for]="checkInPicker">`, `<mat-datepicker #checkInPicker>`. Add `<mat-error>` for required.
     - Check-out: `<mat-form-field>` with `<mat-label>Check-out</mat-label>`, `<input matInput [matDatepicker]="checkOutPicker" formControlName="checkOut" [min]="minCheckOutDate">`, `<mat-datepicker-toggle matIconSuffix [for]="checkOutPicker">`, `<mat-datepicker #checkOutPicker>`. Add `<mat-error>` for required.
   - **Row 3: Guests + Rooms** (in `.form-row`)
     - Guests: `<mat-form-field>` with `<mat-select formControlName="guests">` and `@for (num of [1,2,3,4,5,6,7,8]; track num) { <mat-option [value]="num">{{ num }} {{ num === 1 ? 'Guest' : 'Guests' }}</mat-option> }`
     - Rooms: `<mat-form-field>` with `<mat-select formControlName="rooms">` and `@for (num of [1,2,3,4,5]; track num) { <mat-option [value]="num">{{ num }} {{ num === 1 ? 'Room' : 'Rooms' }}</mat-option> }`
   - **Submit button:** Split into two buttons with @if/@else (same pattern as SearchComponent to avoid Material content projection warnings):
     - Searching state: `<button mat-raised-button color="primary" type="submit" class="search-button" [disabled]="true">` with `<mat-spinner diameter="20">` and "Searching..."
     - Normal state: `<button mat-raised-button color="primary" type="submit" class="search-button" [disabled]="hotelSearchForm.invalid">` with search icon and "Search Hotels"

3. **Results section** — only show when `searchResults().length > 0`:
   - **Results header:** `<h2>{{ sortedHotels().length }} hotels found</h2>`
   - **Sort control:**
     - `<mat-form-field appearance="outline" class="sort-field">` with `<mat-select (selectionChange)="setSortBy($any($event).value)" [value]="sortBy()">`:
       - `<mat-option value="price">Price (lowest)</mat-option>`
       - `<mat-option value="rating">Rating (highest)</mat-option>`
   - **Hotel result cards** — `@for (hotel of sortedHotels(); track hotel.id)`:
     - `<mat-card class="hotel-card">`
     - `<mat-card-content>`:
       - `.hotel-header` row: hotel name (`hotel.name`) and price per night (`hotel.pricePerNight.total | currency:hotel.pricePerNight.currency` + "/night" label)
       - `.hotel-rating` row: star icons using `renderStars(hotel.rating)` — `@for (star of renderStars(hotel.rating); track $index) { <mat-icon class="star-icon">{{ star }}</mat-icon> }` followed by `<span class="rating-text">{{ formatRating(hotel.rating) }}</span>`
       - `.hotel-details` row: address with location_on icon
     - `<mat-card-actions>`:
       - "Add to Itinerary" button: `<button mat-button color="primary" (click)="addToItinerary(hotel)">` with add icon
       - Provider link: `<a mat-button [href]="hotel.link.url" target="_blank" rel="noopener noreferrer">` with open_in_new icon and "View on {{ hotel.link.provider }}"

4. **Empty state** — show when `searchResults().length === 0 && !isSearching() && hasSearched()`:
   - `<mat-card class="empty-state">` with hotel icon and "No hotels found. Try a different destination or dates."

Use Angular's `@if`, `@for` control flow (NOT `*ngIf`, `*ngFor`). Use `@for` with `track` clause.
