  <files>triply/src/app/core/api/hotel.mapper.ts</files>
  <action>
Create `triply/src/app/core/api/hotel.mapper.ts` with:

1. **BookingComHotel interface** (the external API shape from booking-com15.p.rapidapi.com):
   ```typescript
   export interface BookingComHotel {
     hotel_id: number;
     hotel_name?: string;
     hotel_name_trans?: string;
     latitude?: number | string;
     longitude?: number | string;
     address?: string;
     address_trans?: string;
     review_score?: number;       // 0-10 scale from Booking.com
     min_total_price?: number;
     composite_price_breakdown?: {
       gross_amount?: {
         value?: number;
         currency?: string;
       };
     };
     currency_code?: string;
     url?: string;
     checkin?: { until?: string; from?: string };
     checkout?: { until?: string; from?: string };
   }
   ```
   Mark most fields as optional (`?`) because API responses vary. The mapper will use fallbacks for missing values.

2. **HotelMapper class** — `@Injectable({ providedIn: 'root' })`, implements `Mapper<BookingComHotel, Stay>`:
   - `mapResponse(raw: BookingComHotel, checkIn?: string, checkOut?: string): Stay` — transforms one Booking.com hotel to the canonical Stay model:
     - `name` = `raw.hotel_name_trans || raw.hotel_name || 'Unknown Hotel'`
     - `location` = `{ latitude: parseFloat(String(raw.latitude || 0)), longitude: parseFloat(String(raw.longitude || 0)) }`
     - `address` = `raw.address_trans || raw.address || ''`
     - `checkIn` = `checkIn || ''` (passed from search params, not from API response)
     - `checkOut` = `checkOut || ''` (passed from search params, not from API response)
     - `pricePerNight` = compute from raw:
       - `total`: Get the total price from `raw.min_total_price` or `raw.composite_price_breakdown?.gross_amount?.value` or 0. Then divide by number of nights (computed from checkIn/checkOut). If nights = 0 or dates invalid, use the total as-is.
       - `currency`: `raw.currency_code || raw.composite_price_breakdown?.gross_amount?.currency || 'USD'`
     - `rating` = `raw.review_score != null ? Math.round((raw.review_score / 2) * 10) / 10 : null` (convert 0-10 to 0-5 scale, round to 1 decimal)
     - `link` = `{ url: raw.url || 'https://www.booking.com', provider: 'Booking.com' }`
     - `id` = `String(raw.hotel_id)`
     - `source` = `'hotel'`
     - `addedToItinerary` = `false`

   - `private calculateNights(checkIn: string, checkOut: string): number` — utility:
     - Parse both as Date, return `Math.max(1, Math.round((checkOutDate - checkInDate) / (1000 * 60 * 60 * 24)))`
     - If either date is empty or invalid, return 1 (avoids division by zero)

   NOTE: The Mapper interface defines `mapResponse(raw: TExternal): TInternal` with a single parameter. TypeScript allows implementations to add extra optional parameters while still satisfying the interface. The mapper signature is:
   ```typescript
   mapResponse(raw: BookingComHotel, checkIn?: string, checkOut?: string): Stay {
     const ci = checkIn || '';
     const co = checkOut || '';
     // Use ci and co for dates
     ...
   }
   ```
   This keeps mapResponse as a pure function (no stateful setDates() pattern) while satisfying the Mapper<BookingComHotel, Stay> interface.

Import Stay from `../models/trip.models` and Mapper from `./mapper.interface`.

**Critical: Do NOT convert datetime strings to Date objects in the final Stay model.** checkIn/checkOut stay as YYYY-MM-DD strings per project decision (03-01). Date objects are only used internally in calculateNights().

     - BaseApiService.get() prepends the endpoint URL (`/api/hotels` from getEndpoint('hotel')), so calling `this.get('/api/v1/hotels/searchDestination')` produces URL `http://localhost:4200/api/hotels/api/v1/hotels/searchDestination`.      - Pipe through `withBackoff()` for rate limit retry
     - Map response: Extract array from `response.data` (the API returns `{ data: [...] }`). For each item: `{ destId: item.dest_id, name: item.name || item.city_name || '', label: item.label || item.name || '', searchType: item.search_type || 'CITY' }`
     - `catchError(() => of([]))` — destination autocomplete must never surface errors (same pattern as searchAirports)

   - **`searchHotels(params: HotelSearchParams): Observable<ApiResult<Stay[]>>`**:
     - Call `this.get<any>('/api/v1/hotels/searchHotels', { dest_id: params.destId, search_type: params.searchType, checkin_date: params.checkIn, checkout_date: params.checkOut, adults: params.adults, room_qty: params.rooms })`
     - Pipe through `withBackoff()` for rate limit retry
     - Map response: Extract results from `response.data?.result` or `response.data?.hotels` or `[]` (be defensive — the exact field name may vary). Pass checkIn/checkOut dates to mapper:
       ```typescript
       map((response): ApiResult<Stay[]> => {
         const results = response.data?.result || response.data?.hotels || response.data || [];
         const hotels = Array.isArray(results) ? results : [];
         return {
           data: hotels.map((hotel: BookingComHotel) => this.mapper.mapResponse(hotel, params.checkIn, params.checkOut)),
           error: null,
         };
       })
       ```
     - `catchError((error: AppError): Observable<ApiResult<Stay[]>> => of({ data: [], error }))`
