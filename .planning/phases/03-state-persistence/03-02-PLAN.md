  <action>
Create `triply/src/app/core/services/trip-state.service.ts` as an `@Injectable({ providedIn: 'root' })` service.

Import from `@angular/core`: `Injectable`, `signal`, `computed`, `effect`, `inject`.
Import `LocalStorageService` from `./local-storage.service`.
Import all model types from `../models/trip.models`: `Trip`, `Flight`, `Stay`, `CarRental`, `Transport`, `Activity`, `Attraction`, `ItineraryItem`.

**Constants:**
- `STORAGE_KEY = 'triply_trip'` (const, not exported)
- `DEFAULT_TRIP: Trip` — a factory-created default with: `id: crypto.randomUUID()`, `name: ''`, `destination: ''`, `dates: { start: '', end: '' }`, all arrays empty, `createdAt: new Date().toISOString()`, `updatedAt: new Date().toISOString()`. Make this a `const` — it is used as fallback when localStorage is empty or corrupt.

**Service structure:**

1. **Inject `LocalStorageService`** via `inject()` as a private readonly field.

2. **Private WritableSignal** — `_trip = signal<Trip>(this.storage.get<Trip>(STORAGE_KEY) ?? { ...DEFAULT_TRIP })`. This is the startup recovery: the signal initializer reads synchronously from localStorage at construction time. If the key is missing or JSON is malformed, `get()` returns null and the fallback spreads DEFAULT_TRIP. No APP_INITIALIZER needed.

3. **Public readonly signal** — `trip = this._trip.asReadonly()`. Components read this, never `_trip` directly.

4. **Computed slices** — one for each array in the Trip:
   - `flights = computed(() => this._trip().flights)`
   - `stays = computed(() => this._trip().stays)`
   - `carRentals = computed(() => this._trip().carRentals)`
   - `transports = computed(() => this._trip().transports)`
   - `activities = computed(() => this._trip().activities)`
   - `attractions = computed(() => this._trip().attractions)`
   - `itineraryItems = computed(() => this._trip().itineraryItems)`
   - `hasItems = computed(() => this._trip().itineraryItems.length > 0)`

5. **Constructor effect** — `effect(() => { this.storage.set(STORAGE_KEY, this._trip()); })`. This reads `_trip()` registering it as a dependency, so the effect re-runs on every change. Runs as a root effect (microtask-scheduled before any component check). Do NOT pass `allowSignalWrites` — it is deprecated in Angular 19 and a no-op in Angular 21.

6. **Mutation methods** — all follow the same pattern: `this._trip.update(t => ({ ...t, [field]: [newValue], updatedAt: new Date().toISOString() }))`. Implement:
   - `setTripMeta(name: string, destination: string, dates: { start: string; end: string }): void`
   - `addFlight(flight: Flight): void` / `removeFlight(id: string): void`
   - `addStay(stay: Stay): void` / `removeStay(id: string): void`
   - `addCarRental(car: CarRental): void` / `removeCarRental(id: string): void`
   - `addTransport(transport: Transport): void` / `removeTransport(id: string): void`
   - `addActivity(activity: Activity): void` / `removeActivity(id: string): void`
   - `addAttraction(attraction: Attraction): void` / `removeAttraction(id: string): void`
   - `addItineraryItem(item: ItineraryItem): void` / `removeItineraryItem(id: string): void`
   - `updateItineraryItem(updated: ItineraryItem): void` — replaces the item with matching `id` via `.map()`
   - `resetTrip(): void` — sets `_trip` to a fresh DEFAULT_TRIP (new UUID, new timestamps) and calls `this.storage.remove(STORAGE_KEY)` to clear localStorage

   Each `add*` method spreads the existing array with the new item: `[...t.flights, flight]`.
   Each `remove*` method filters by id: `t.flights.filter(f => f.id !== id)`.

**Implementation notes:**
- Do NOT use `allowSignalWrites: true` on effect() — deprecated in Angular 19, no-op in Angular 21
- Do NOT use APP_INITIALIZER — signal initializer provides synchronous hydration
- Do NOT store Date objects — all timestamps are ISO 8601 strings
- The `_trip` field MUST be `private readonly` — no component can call `.set()` or `.update()` directly
