Create two placeholder standalone components using the Angular CLI:

```bash
cd triply
npx ng generate component features/search --standalone --skip-tests
npx ng generate component features/itinerary --standalone --skip-tests
```

If `ng generate` does not accept `--standalone` (it may be the default in v19+), omit it. The key requirement is that the generated components have `standalone: true` in their decorator.

Update each placeholder component template to show a heading indicating what it is:

**search.component.html:**
```html
<h1>Search</h1>
<p>Search for flights, hotels, and more.</p>
```

**itinerary.component.html:**
```html
<h1>Itinerary</h1>
<p>Your trip itinerary will appear here.</p>
```

Now update `app.routes.ts` with lazy-loaded routes:

```typescript
import { Routes } from '@angular/router';

export const routes: Routes = [
  { path: '', redirectTo: 'search', pathMatch: 'full' },
  {
    path: 'search',
    loadComponent: () =>
      import('./features/search/search.component').then(m => m.SearchComponent),
  },
  {
    path: 'itinerary',
    loadComponent: () =>
      import('./features/itinerary/itinerary.component').then(m => m.ItineraryComponent),
  },
  { path: '**', redirectTo: 'search' },
];
```

Build again to ensure the lazy routes resolve correctly. Then serve the app briefly with `npx ng serve --port 4200` to confirm navigation works (visit `/search` and `/itinerary`).
